## Template 1: Product Manager PRD Generator

**Source**: [awesome-chatgpt-prompts](https://github.com/f/awesome-chatgpt-prompts)  
**Category**: Requirements Documentation  
**Quality Score**: 9/10

### Prompt Structure
```
Act as: Product Manager
Task: Write PRD with structured headers
Output: Subject, Introduction, Problem Statement, Goals and Objectives, 
        User Stories, Technical requirements, Benefits, KPIs, 
        Development Risks, Conclusion
```

### Strengths
- Comprehensive PRD structure
- Covers all essential sections
- Clear output format
- Includes risk assessment

### Weaknesses
- Generic, not e-commerce specific
- No prioritization framework

### Use Cases
- Initial product planning
- Feature requirement documentation
- Stakeholder alignment

### Example Input
```
Subject: E-commerce checkout optimization
```

### Example Output
```
Subject: E-commerce Checkout Optimization

Introduction: This PRD outlines improvements to reduce cart abandonment...

Problem Statement: Current checkout has 68% abandonment rate...

Goals and Objectives:
- Reduce abandonment to <40%
- Improve mobile UX
- Add guest checkout

User Stories:
- As a customer, I want to checkout without creating account
- As a mobile user, I want autofill payment info

Technical Requirements:
- Payment gateway integration (Stripe, PayPal)
- Address validation API
- Mobile-responsive forms

Benefits:
- Increased conversion rate
- Better user experience
- Competitive advantage

KPIs:
- Cart abandonment rate
- Checkout completion time
- Mobile conversion rate

Development Risks:
- Payment gateway downtime
- PCI compliance requirements
- Cross-browser compatibility

Conclusion: Implementation timeline 6 weeks...
```

## Template 2: Startup Idea Generator (Business Planning)

**Source**: [awesome-chatgpt-prompts](https://github.com/f/awesome-chatgpt-prompts)  
**Category**: Business Model Canvas  
**Quality Score**: 8/10

### Prompt Structure
```
Input: People's wish/problem
Output: Complete business plan with:
- Idea name
- One-liner
- Target user persona
- Pain points to solve
- Value propositions
- Sales & marketing channels
- Revenue streams
- Cost structures
```

### Strengths
- Holistic business model view
- Covers revenue and costs
- User-centric approach
- Market positioning

### Weaknesses
- High-level, lacks implementation details
- No competitive analysis

- Pitch deck preparation
- Business model validation

People want: Sustainable fashion shopping with transparency

One-liner: Transparent supply chain for ethical fashion

Sales & Marketing Channels:
- Instagram/TikTok influencer partnerships
- SEO content marketing
- Email nurture campaigns
- Referral program

Revenue Streams:
- Product sales (70%)

- B2B white-label platform (10%)

Cost Structures:
- Manufacturing: 35%

- Technology: 15%

## Template 4: Developer Relations Consultant (Competitive Analysis)

**Source**: [awesome-chatgpt-prompts](https://github.com/f/awesome-chatgpt-prompts)  
**Category**: Market Research  
**Quality Score**: 8/10

### Prompt Structure
```
Act as: Developer Relations consultant
Task: Research package + documentation
Analysis: Quantitative (StackOverflow, Hacker News, GitHub)
Output: Competitor comparison, benefits/shortcomings
```

### Strengths
- Data-driven approach
- Multiple data sources
- Comparative analysis
- Quantitative metrics

### Weaknesses
- Tech-focused (not business-focused)
- Requires adaptation for e-commerce

### Use Cases
- Technology stack selection
- Vendor evaluation
- Platform comparison

### Example Input (Adapted for E-commerce)
```
Compare Shopify vs WooCommerce vs custom Next.js solution
```

### Example Output
```
Competitive Analysis: E-commerce Platforms

Data Sources:
- GitHub stars/forks
- Reddit discussions (r/ecommerce)
- G2 reviews
- Market share data

Shopify:
+ Pros:
  - Fastest time-to-market (days)
  - Built-in payment processing
  - App ecosystem (8000+ apps)
  - Managed hosting
  - PCI compliance included
- Cons:
  - Transaction fees (2% unless Shopify Payments)
  - Limited customization
  - Vendor lock-in
  - Monthly costs ($29-$299+)
- Best for: Quick launch, non-technical teams

WooCommerce:
+ Pros:
  - Free core plugin
  - Full customization (WordPress)
  - No transaction fees
  - Large community
- Cons:
  - Requires hosting management
  - Security responsibility
  - Performance optimization needed
  - Plugin compatibility issues
- Best for: WordPress users, budget-conscious

Custom Next.js:
+ Pros:
  - Complete control
  - Optimal performance
  - Modern tech stack
  - No platform fees
- Cons:
  - Longest development time (months)
  - Requires dev team
  - Maintenance overhead
  - Build everything from scratch
- Best for: Unique requirements, scale

Recommendation: 
- MVP → Shopify
- Content-heavy → WooCommerce
- Custom features → Next.js
```
